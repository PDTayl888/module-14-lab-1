1.  Efficiency. With a well implemented CI/CD pipeline there is no confusion about how things are done. As long as everyone follows the prescribed pipeline and hopefully the pipeline is designed in the most efficient manner then all doubts are gone and everyone can simply be in flow state. Time saved is money made. A well designed pipeline wil also provide for optimal quality and security which will ultimately lead to less mistakes and less unhappy customers which leads to more money and less stress for the company and hopefully the team.
---------------

2.  Continuous integration is the practice testing and frequently merging code into a main code branch. This is done in small chinks and frequently so it simply isn't too complicated and overwhelming to test and integrate. Once a set of tests are passed in the CI part of the pipeline then the code is automatically deployed in what is known as the continuous deployment phase of the pipeline.
-----------------

3.  As a small start up with limited budget I recommend Github Actions because it has a generous free tier and it is highly likely your codebase is already there. It also seems the easiest to set up by adding a single .github/workflow file.
-----------------

4.  The customizability of Jenkins would especially be helpful in an industry that is highly regulated like a bank or other financial institution. They can use custom Groovy scripts to deal with the mandated security regulations. Plus Jenkins has a large eco system of plugins to deal streamline processes that are common in nearly an situation.
------------------
